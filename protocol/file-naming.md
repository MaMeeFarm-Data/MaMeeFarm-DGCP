# DGCP File Naming Rules

Clear naming makes data readable for humans and predictable for AI.

### 1. Use lowercase
Example:
po-life-2025-01-01.json

### 2. Use hyphens instead of spaces
Good:
duck-feeding-2025-01-01.json
Bad:
duck feeding 2025 JAN 1.JSON

### 3. Date-first format (YYYY-MM-DD)
This keeps files sorted in the correct order.

### 4. No special characters
Avoid:
! @ # $ % ^ & * ( ) [ ]

### 5. Keep names simple and descriptive
Examples:
polife-morning-check-2025-01-05.json
polabor-cleaning-coop-2025-01-03.json
pohw-unit-2025-01-10.json


Naming rules keep DGCP transparent and machine-friendly.
